Reasoning about layered allocators

User view

  • Heap memory can be inaccessible or user-owned
  • malloc makes some inaccessible memory user-owned (or fails)
  • free makes some user-owned memory inaccessible
  • Accessing inaccessible memory is undefined behavior
  • Memory is managed in nonoverlapping units called allocations

Allocator view

  • Heap memory can be inaccessible, allocator-owned, or user-owned
  • malloc makes some inaccessible memory user-owned (or fails)
  • It also creates some allocator-owned memory for tracking
  • free makes some user-owned memory allocator-owned
  • Accessing inaccessible memory is undefined behavior
  • Modifying user-owned memory will confuse the user
